# pstack glossary

Key terms used across pstack skills, rubrics, and challenge seeds. Sorted alphabetically.

---

**Artifact** — A structured markdown file produced by a pstack generator skill. Numbered by phase: `00-vision.md`, `02-prd.md`, `03-design-brief.md`.

**Artifact schema version** — The version of the `.pstack-meta.json` format, independent of the pstack tool version. Allows future migrations.

**Calibration set** — Three scored reference artifacts per artifact type (score-2, score-3.5, score-5) that anchor Claude's rubric scoring to consistent numeric benchmarks.

**Challenge** — A CEO/CPO-level provocation generated by `/challenge`, grounded in the user's current project artifacts and user profile. Appended to `CHALLENGES.md`.

**Common flags** — Anti-patterns a persona reliably surfaces when reviewing product work. Defined in `persona.yaml`.

**Dependency quality gate** — A check run before a generator skill that validates the precondition artifact (e.g., vision) meets the rubric passing floor. Blocks execution if quality is too low.

**Design brief** — A structured document translating a PRD into design direction: user & job, primary flow, screen inventory, interaction principles, visual/voice constraints.

**Four risks (Cagan)** — Value risk (do users want it?), Usability risk (can users use it?), Feasibility risk (can we build it?), Viability risk (does it work for the business?).

**ikigai** — Japanese concept for the convergence of four circles: what you love, what you're good at, what the world needs, and what you can be paid for. Central to the Bhaskar persona.

**Internal critique loop** — The draft→self-critique→revise cycle that runs inside every generator skill before the output is surfaced to the user.

**JTBD (Jobs to Be Done)** — A framework treating products as things people "hire" to do a job in their life. Focus on the job, not the user persona.

**Lazy loading** — The practice of reading `knowledge/INDEX.md` first, then pulling only the specific framework files flagged as relevant — rather than loading the entire knowledge base.

**Lens** — A persona's characteristic perspective when reviewing product work. Defined as a list in `persona.yaml`.

**LNO (Leverage / Neutral / Overhead)** — Shreyas Doshi's task classification framework: Leverage tasks multiply impact; Neutral tasks maintain; Overhead tasks drain without creating value.

**North Star metric** — The single metric that best captures whether customers are getting the core value from the product. Ideally a leading indicator of long-term retention.

**Passing floor** — The weighted average rubric score (default: 3.5/5) below which an artifact is not ready to advance to the next skill. Enforced by the dependency quality gate.

**Persona** — A structured `persona.yaml` file defining how a product-leader voice reviews artifacts. Adding a new reviewer requires only one file drop (no code changes).

**PRD (Product Requirements Document)** — A structured document defining the problem hypothesis, user stories, acceptance criteria, success metrics, scope, risks, and milestones for a product slice.

**pstack-workspace** — The directory in the user's current project where all artifacts and reviews are written (`./pstack-workspace/<slug>/`).

**Rubric** — A structured YAML file defining scored dimensions for an artifact type. Used by `/critique` and the internal quality loop. Lives in `knowledge/rubrics/`.

**Self-critique** — The internal step in which a generator skill scores its own draft against the rubric before presenting it to the user.

**Signature questions** — The characteristic questions a persona asks when reviewing product work. Defined in `persona.yaml`.

**Slug** — A URL-safe project identifier (lowercase, hyphens) used as the directory name inside `pstack-workspace/`. Derived from the product name.

**State machine** — The set of project states tracked in `.pstack-meta.json`: `DRAFT_VISION → VISION_DONE → DRAFT_PRD → PRD_DONE → DRAFT_BRIEF → BRIEF_DONE`.

**TL;DR header** — Three lines of `tldr` frontmatter at the top of every framework file, enabling skills to skip the full file when a quick summary is sufficient.

**User profile** — `~/.config/pstack/user-profile.yaml` — global identity seeded by `/onboard`. Contains ikigai, values, customer archetypes. Read by `/challenge` and persona reviews for personalization.

**Vision doc** — The first artifact in the pstack loop. Covers: problem, target user & JTBD, vision statement, why now, 3-year success picture, non-goals, North Star metric, key risks.
